# Decentrathon-inDrive-C.A.T.S

Decentrathon inDrive — кейс 2: скоринг сельхозпроизводителей (субсидии).

Прототип **merit-based** ранжирования заявок из реестра ИСС `subsidy.plem.kz`: модель оценивает вероятность положительного исхода по историческим статусам, UI формирует **shortlist** для комиссии и показывает **топ-3 фактора (SHAP)**. Финальное решение за человеком; модель не заменяет эксперта (см. ТЗ).

## Данные

- Исходный файл: `Subsidy Report 2025.xlsx` (выгрузка реестра заявок).
- **Целевая переменная `outcome_positive`** (обучение только на строках, где статус однозначен):
  - **1 (положительный исход):** `Исполнена`, `Одобрена`, `Сформировано поручение`, `Получена`.
  - **0 (отказ):** `Отклонена`.
  - **Исключено из обучения:** `Отозвано` (заявитель отозвал заявку; не классифицируем как успех/провал).

Признаки:

- **Из выгрузки:** даты/сезонность, коды региона/района/направления/программы, норматив и суммы, лог суммы, отношение суммы к нормативу, агрегаты по региону (средняя сумма, число заявок), флаг возможных повторных подач в тот же день по району.
- **Симулированные** (в реальной интеграции пришли бы из ветреестра, ГБД ФЛ, земельного кадастра и т.д.; здесь детерминированно от `application_id` для демо): опыт, площадь, поголовье, история субсидий, утилизация, вет/продавец/покупатель, документы, риск фрода, налог, земля на голову, эвристический `merit_index_heuristic`.

Подробнее о колонках — в `scoring/features.py`.

## Запуск

```bash
cd decentrathon
python3 -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

### Обучение модели

```bash
export PYTHONPATH=.   # Windows: set PYTHONPATH=.
python -m scoring.train
```

Артефакты: `artifacts/xgb_model.joblib`, `feature_columns.joblib`, `training_meta.json`, `feature_importance.json`.

### UI (Streamlit)

```bash
PYTHONPATH=. streamlit run app.py
```

В боковой панели: фильтр по области и программе, размер shortlist, порог score. Ниже — таблица и **SHAP-топ-3** для выбранной заявки.

## Ограничения и честность демо

- Метка **историческая** (статус заявки), а не «качество хозяйства» вне процедуры; интерпретация score — «согласованность с прошлыми паттернами одобрения в данных».
- Симулированные признаки **не** подменяют реальные реестры; их роль — показать архитектуру и explainability.
- Агрегаты по региону считаются на всей выборке при инференсе; в проде лучше считать только на train или по скользящему окну.

## Демо-видео (сценарий для съёмки, 2–3 минуты)

1. Проблема FIFO vs merit (1 слайд / устно).
2. Запуск `streamlit run app.py`, показать shortlist по области.
3. Выбрать заявку — показать SHAP топ-3.
4. Показать `training_meta.json` / метрики на hold-out.
5. Оговорить: решение комиссии окончательное, модель — помощник.
